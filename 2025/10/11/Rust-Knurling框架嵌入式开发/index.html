本章使用 Knurling Rust 嵌入式框架完成环境部署, 快速使用, 实机运行等任务

什么是Knurling

Knurling 是 Ferrous Systems 公司的一个项目宗旨是改善Rust的嵌入式开发体验, 为此他们开发了一些列的工具去实现这一目标.
目前 Rust 嵌入式开发官方 已经推荐我们使用这个框架了

Knurling有什么功能

相对于官方的框架目前 Knurling 在其基础上支持了

  • 近似 Rust 的开发体验 (老框架和Rust开发流程有些割裂)
  • 支持测试

环境部署

  1. Rust 安装 略过
  2. 安装 flip-link:
    cargo install flip-link
  3. 安装 probe-rs:
    curl --proto '=https' --tlsv1.2 -LsSf https://github.com/probe-rs/probe-rs/releases/latest/download/probe-rs-tools-installer.sh | sh
  4. 安装 cargo-generate:
    cargo install cargo-generate

  1. 设置 probe-rs
    可以使用 probe-rs chip list 来列出 probe-rs 支持的单片机, 并填写进 .cargo/config.toml
  1. 更换目标编译器
    在 .cargo/config.toml 中切换目标编译器, 更具具体的单片机架构切换到对应的编译器, 编译器通过 rustup 安装

由于STM32H7是Cortex-M7架构, 所以我们要切换到对应编译器(Cortex-M7F后面的F代表的是使用FPU浮点运算器)

  1. 添加hal库

  1. 导入 hal 库

更具实际单片机型号调整注释

The memory.x file is included in the cortex-m-rt linker script link.x, and so link.x is the one you should tell rustc to use (see the .cargo/config.toml file where we do that).

  1. 运行
    现在你可以通过 cargo run 来运行 src/bin 中的示例了, 这里要注意, cargo并不是根据文件名去编译的, 而是通过Cargo.toml中定义的[[bin]]编译, 只是默认和文件同名而已, 不要混淆.

运行这个程序和进行单元测试并不能脱离单片机运行, 如果想脱离单片机运行测试要配置qemu虚拟机运行
