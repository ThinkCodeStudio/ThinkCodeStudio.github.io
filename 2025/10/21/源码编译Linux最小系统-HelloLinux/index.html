发表于 2025-10-21 分类于 Linux
本文字数： 867 阅读时长 ≈ 3 分钟

前言

一直想踏入Linux这个级别的嵌入式开发, 想使用性能更强的硬件设备, 以前也尝试过研究 rk3506 的SDK, 但是内容是在太多了, 也非常复杂, 而且瑞芯微和板商都做了非常多的工作导致非常完善无从下手, 所以我尝试编译Linux源码来起步学习.

需要的资源

  • linux 源码
  • busybox 源码
  • qemu 源码

实验平台

  • CPU: i5-8250u
  • OS: Ubuntu24.04

参考

  1. https://zhuanlan.zhihu.com/p/683380725
  2. https://blog.csdn.net/benkaoya/article/details/129469116

Linux 编译

使用 git 下载 Linux 源码(6.18-rc2)

linux源码比较大, 需要一些时间. 从github拉取下来的主线代码是最新发布版

安装一些依赖

然后在linux源码的根目录下运行编译

编译完成后会显示编译出 bzImage 的路径, 这个就是 linux 固件, 它可以使用 qemu 直接运行, 不过会报没有根目录的错误

编译 busybox

busybox是一套工具集, 用于编译像 ls, cat这些命令

使用 git 下载 源码

在 busybox 根目录下运行

配置

进入 Setting->Build static binary, 勾选这一项
编译

编译过程中可能 tc.c 文件出现错误, 这个是个BUG, 貌似只有编译静态库才出现, 动态没有
解决办法简单, 直接移除 tc.c 文件, 不影响本次实验.

参考:
https://blog.csdn.net/2202_75820736/article/details/142054146

安装

运行完成会在busybox的根目录下出现 _install 文件夹, 这就是我们需要的

编译 qemu

我是用的是qemu源码编译出来的工具, 由于apt安装的版本太老, 所以我习惯使用编译的qemu
使用 git 下载 qemu源码
