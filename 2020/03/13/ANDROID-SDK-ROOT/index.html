Android Studio 3.6 错误Emulator:PANIC:Cannot find AVD system path. Please define ANDROID_SDK_ROOT

发表于 2020-03-13 更新于 2020-04-29 分类于 IDE 阅读次数： ℃ Valine：
本文字数： 744 阅读时长 ≈ 1 分钟
  • AS错误错误解决记录

每次升级AS就会面临错误错误错误。。。

前几个版本也有大量的错误，尤其是添加fultter，为了能用AS开发我折腾了好几周，对于小白开发Android，配置开发环境就劝退了大波人，谁不想下载完软件直接开发，但是它不像VS那样人性化，使用AS就要面对各种各样的软件错误。前几个版本我先不整理了，我也没记住，等到我换电脑时补充，现在开始我要追踪IDE错误的解决方法。

  • 系统平台：windows10

在升级到AS 3.6时运行AVD虚拟机就会在窗口显示这样的信息

  • 这是找不到AVD的路径了，具体我不知道，我就理解为AS把SDK和AVD分开了。
  • 解决办法就是环境变量里加一个AVD的变量，变量名ANDROID_SDK_HOME

前提要把原来的虚拟机都删了，由于需要重新定义路径，最后还是要重建虚拟机

然后重启AS就可以运行虚拟机了

OK这就是我的解决方法
十分感谢这位博主原文
博主原文：
为何是设置 ANDROID_SDK_HOME 而不是 ANDROID_SDK_ROOT？ANDROID_SDK_HOME 到底有啥用？可以参考 Android Studio 相关的系统环境变量 里的说明。概况来说，启动模拟器时会从 ANDROID_SDK_HOME 指定的目录查找 AVD 目录（正规来说应该是设置成 ANDROID_AVD_HOME）。估计是 Android Studio 这里写错成了 ANDROID_SDK_ROOT。

后续我会把Java8升级成Java11，如果AS还会报什么错我还会记录。
说实话我真不想折腾了
